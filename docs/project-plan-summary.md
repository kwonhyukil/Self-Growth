## 항목별 작성 방향

### 1. 프로젝트 개요
- Self Growth Log는 감정 기록과 일본어 자기표현 학습을 결합한 자기성장 지원 웹 서비스로 설명한다.
- 기록, AI 피드백, 수정, 성장 확인이 하나의 흐름으로 이어지는 서비스라는 점을 강조한다.

### 2. 추진 배경 및 필요성
- 기존 감정 기록 서비스는 성장 연결이 약하다는 점을 설명한다.
- 기존 언어 학습 서비스는 실제 자기 경험 표현 훈련이 부족하다는 점을 설명한다.
- 두 영역을 연결한 서비스의 필요성을 제시한다.

### 3. 문제 정의
- 감정 기록이 자기성장 데이터로 이어지지 않는 문제
- 외국어 학습이 실제 자기표현 훈련으로 연결되지 않는 문제
- AI 피드백이 있어도 수정 이력과 개선 정도를 누적 관리하기 어려운 문제

### 4. 목표 및 기대효과
- 감정 기록과 일본어 자기표현 학습의 결합
- AI 피드백 기반 반복 학습 경험 제공
- 수정 이력과 성장 통계를 통한 변화 가시화
- 자기이해와 언어 능력 향상 동시 지원

### 5. 서비스 내용
- 사용자가 감정과 사건을 기록한다.
- 자기 칭찬 문장을 한국어와 일본어로 작성한다.
- AI가 일본어 문장을 점검한다.
- 사용자가 피드백을 반영해 문장을 수정한다.
- 언어화 기능으로 생각을 더 깊이 정리한다.
- 결과를 대시보드와 통계로 확인한다.

### 6. 주요 기능
- 사용자 인증
- 감정 로그 관리
- 일본어 점검
- Rewrite
- 언어화
- 대시보드 및 통계

### 7. 사용 기술 스택
- Frontend: React, TypeScript, Vite, React Router, TanStack Query, Tailwind CSS
- Backend: Node.js, Express, Prisma, JWT, Zod
- Database: MySQL
- Infra: Docker Compose
- AI: OpenAI API

### 8. 개발 일정
- 기획 -> 설계 -> 핵심 기능 구현 -> 확장 기능 구현 -> 테스트 및 보완 -> 문서화 순으로 작성한다.
- 학습은 별도 일정으로 분리하지 않고 각 단계에 필요한 기술 학습을 병행하는 방식으로 적는다.
- 예시 흐름:
  - 1주차: 기획 및 요구사항 분석, React/Vite 기본 구조 학습
  - 2주차: 화면 및 데이터 설계, Prisma/MySQL 학습
  - 3주차: 인증 및 백엔드 기본 구조 구현, JWT/Zod 학습
  - 4주차: 로그 CRUD 구현, API 연동 학습
  - 5주차: AI 피드백 및 Rewrite 구현, OpenAI API 연동 학습
  - 6주차: 언어화, 대시보드, 통계 기능 구현
  - 7주차: 테스트 및 보완
  - 8주차: 문서화 및 최종 정리

## 추가 메모

- 계획서에서는 화면 구성도, 시스템 구성도, 상세 DB 설계는 필수 항목이 아니므로 제외해도 무방하다고 정리했다.
- 현재 선택한 8개 항목만으로도 계획서 구조는 충분히 자연스럽다.
- 기술 스택은 개발 일정 앞에 두는 흐름이 더 자연스럽다고 정리했다.
